# Plan: Convert Timeline Editor to Vanilla JS with Vite

## Goal
Transform the provided timeline editor HTML (with inline CSS/JS) into a modern Vite project using vanilla JavaScript, maintaining all functionality while improving maintainability and performance.

## Current State Analysis
- Existing app: Vue.js frontend in `apps/director/frontend/`
- Provided code: Single HTML file with inline styles and JavaScript
- Requirements: Use Vite and vanilla JS (no frameworks)

## Implementation Plan

### Phase 1: Project Structure Setup
1. **Clean existing Vue dependencies**
   - Remove Vue-related packages from package.json
   - Remove Vue-specific files (.vue, router, etc.)
   - Keep Vite and essential build tools

2. **Create vanilla JS structure**
   - `src/index.html` - Main HTML template
   - `src/main.js` - Entry JavaScript file
   - `src/styles.css` - Extracted CSS styles
   - `src/components/` - Modular JavaScript components (if needed for organization)

### Phase 2: Code Separation and Modularization
1. **Extract CSS**
   - Move all inline styles to `src/styles.css`
   - Ensure CSS variables and responsive design are preserved
   - Optimize for Vite's CSS handling

2. **Extract JavaScript**
   - Move inline script to `src/main.js`
   - Refactor into ES6 modules for better organization:
     - `state.js` - Application state management
     - `render.js` - Rendering functions
     - `events.js` - Event handlers
     - `utils.js` - Utility functions

3. **Update HTML template**
   - Simplify index.html to use Vite imports
   - Link external CSS and JS files
   - Maintain semantic structure

### Phase 3: Vite Configuration
1. **Update vite.config.ts**
   - Configure for vanilla JS (remove Vue plugin)
   - Set proper entry points
   - Configure build output for the timeline editor

2. **Update package.json**
   - Remove Vue dependencies
   - Ensure vanilla JS build scripts work
   - Add any needed vanilla JS libraries if necessary

### Phase 4: Testing and Optimization
1. **Functionality verification**
   - Test all interactive features (timeline, playback, chat, etc.)
   - Ensure responsive design works
   - Verify toast notifications and state management

2. **Performance optimization**
   - Leverage Vite's HMR for development
   - Optimize bundle size
   - Ensure fast loading

## Tradeoffs and Considerations
- **Modularity vs Simplicity**: Splitting into modules improves maintainability but adds complexity
- **State Management**: Current inline state could be moved to a simple store pattern
- **Component Organization**: For vanilla JS, consider using custom elements or just functions

## Questions for User
1. Should we maintain the existing project structure or create a clean vanilla JS setup?
2. Do you want to keep any Vue-specific features or fully replace with vanilla?
3. Any specific performance or bundle size requirements?
